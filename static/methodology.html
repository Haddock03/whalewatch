Méthodologie

L'argument du produit, c'est pas de boîte noire. Cette page documente d'où viennent les chiffres, comment le Smart Money Score est calculé, à quelle fréquence les données sont rafraîchies, et ce que WhaleWatch ne voit pas.

Sources D'où viennent les données

WhaleWatch agrège trois sources publiques. Aucune donnée n'est inventée ni interpolée — un chiffre absent reste absent.

Dune Analytics

  • Top wallets DEX — volume cumulé par adresse sur les 7 derniers jours, tous DEX Ethereum.
  • Patterns de trading — taille moyenne, hold time, paires les plus tradées, distribution horaire.

Etherscan API

  • Name tags — pour identifier CEX, bridges, routers, market makers connus.
  • Détection EOA vs contrat (eth_getCode).

Flashbots / MEV-Boost

  • Flashbots transparency dashboard — pour caractériser les MEV bots (sandwich, arbitrage atomique).
  • EigenPhi — référence externe pour les attaques sandwich.

CoinGecko

  • Prix ETH/BTC live pour les KPI affichés. Aucune utilisation pour le scoring.

Score Formule du Smart Money Score

Le score est sur 100. Il combine volume, taille moyenne par trade, diversité de tokens, et applique des pénalités pour les comportements qui ne sont pas de l'alpha (MEV, spam, infrastructure).

score = 8 + volume_tier + avg_trade_tier + diversity + eoa_bonus + activity_bonus + net_eth_bonus
        − mev_penalty − spam_penalty − concentration_penalty − infra_penalty

Détails des pénalités

  • MEV : −45 si la catégorie est MEV Bot (sandwich, frontrun, backrun).
  • Spam : −18 si plus de 5000 trades sur 7j (bot probable) ; −7 si plus de 1500.
  • Concentration : jusqu'à −10 si un seul jour représente plus de 50 % du volume hebdo (pump-chase ou wash trading possible).
  • Infrastructure (nouveauté P1.1 backend) : −45 pour Market Maker, −55 pour CEX hot wallet, Bridge, Router DEX. Voir ci-dessous.

Exclusion infrastructure (P1.1)

Les wallets d'infrastructure ont un volume colossal mais ne représentent pas de l'alpha discrétionnaire. Ils sont désormais explicitement pénalisés côté backend dans le score, avant même qu'il n'arrive au frontend :

  • CEX hot wallets — Binance, Coinbase, Kraken, Bybit, OKX, etc. Détectés par les name tags Etherscan.
  • Bridges — Stargate, Across, Wormhole, Hop, Synapse, deBridge, Orbiter, Li.Fi…
  • Routers DEX — Uniswap, 1inch, 0x, Paraswap, CowSwap, Kyber, Odos, Matcha…
  • Market Makers — Wintermute, Jump Trading, Jane Street, GSR, Cumberland, Flow Traders…
  • MEV Bots — sandwich, frontrun, backrun, arbitrage atomique, flashbot bundles.

Invariant garanti par les tests : même à 1 Md$ de volume, un wallet d'infrastructure obtient un score < 65 (sous le seuil « Solid ») et n'apparaît donc jamais dans le Smart Money Leaderboard.

Code source : wallet_classifier.py (regex), smart_score.py::_infra_penalty (barème), test_scoring.py (37 cas de test). Le module JS classifyWalletType() dans static/index.html est un miroir fidèle pour le filtre UI.

Cluster detection

Une firme de trading propriétaire peut déployer 5 bots différents, chacun avec un volume DEX élevé. Sans détection, ces 5 wallets peuvent monopoliser le leaderboard et donner l'illusion de 5 alphas indépendants alors qu'il n'y en a qu'un — l'entité qui contrôle le déployeur.

WhaleWatch interroge le deployer de chaque smart contract via Etherscan. Si N ≥ 2 contrats partagent un même deployer, ils sont marqués comme cluster (identifié par les 6 premiers chars du deployer, ex. ⛓ 0x858ea9).

  • Dans le tableau principal : badge violet ⛓ 0x… à côté de l'adresse, tooltip indique combien d'autres wallets en font partie.
  • Dans le Smart Money Leaderboard : déduplication automatique — un seul wallet par cluster (le mieux scoré) avec un compteur +N indiquant les siblings cachés.
  • Filtre toolbar « Dédup clusters » applique la même règle à toute la table.
  • Au-delà de 30 wallets sur le même deployer, on suppose une factory partagée (CREATE2 multisig deployers, Safe factories…) plutôt qu'un cluster privé.

Le clic sur un wallet faisant partie d'un cluster affiche la liste de ses siblings dans la modal de détail.

Chaque chain a son cache séparé (cache/results_<chain>.json), son propre Sonar, son propre Smart Money Leaderboard. L'API Etherscan V2 unifie l'enrichissement on-chain (paramètre chainid), et Dune Analytics couvre la table dex.trades sur toutes ces chains.

Les explorer links (Etherscan, Arbiscan, Basescan, Optimistic Etherscan) s'adaptent automatiquement à la chain sélectionnée. Le score, la classification infra et la détection de clusters fonctionnent de manière chain-agnostique.

Les patterns whales (zones BUY/SELL, heatmap horaire, top paires/DEX, distribution des tailles, prix MEV) sont désormais disponibles sur les 4 chains. Note : les seuils internes (e.g. nombre minimum de trades pour qualifier un wallet MEV) restent calibrés sur Ethereum et peuvent être affinés au fil de l'observation sur L2.

Calibration L2 en cours : les seuils de volume du Smart Score (≥10M$ = 22 pts, ≥1Md$ = 40 pts) sont calibrés sur Ethereum. Sur les L2 (volumes typiques 50–100× plus petits), les meilleurs wallets plafonnent autour de score 40. Un facteur d'échelle par chain est prévu pour rétablir des distributions Alpha/Solid/Avg comparables.

Limites Ce que WhaleWatch ne voit pas

WhaleWatch couvre les swaps DEX on-chain EVM (Ethereum + L2 majeurs). Le produit ne voit ni les positions sur les exchanges centralisés (Binance, Coinbase, Bybit…), ni les contrats à terme et perpétuels (Binance Futures, dYdX, GMX, Hyperliquid…).

Concrètement, ça veut dire :

  • Un wallet qui shorte 200M$ sur Binance Futures et dont le hedge spot transite par un bridge sera invisible pour WhaleWatch.
  • Une baleine qui accumule un token sur Coinbase puis withdraw ne génère pas de signal tant qu'elle ne touche pas le DEX.
  • Les flux stablecoin off-chain (rampes fiat ⇄ USDC) sont invisibles.

Le produit reste utile parce qu'une grosse partie du flux opportuniste passe quand même par les DEX (memecoins, tokens long tail, sniping, arbitrage cross-DEX). Mais il faut savoir où la lampe ne porte pas.

Autres limites assumées

  • EVM chains uniquement. Solana, Aptos, Sui, Bitcoin et autres non-EVM sont hors couverture.
  • Pas de filtre 100% fiable contre le wash trading. La pénalité de concentration en réduit l'impact, mais ne l'élimine pas.
  • Les name tags Etherscan ne sont pas exhaustifs. Un wallet d'infrastructure non labellisé peut passer comme « EOA » jusqu'à ce qu'on l'ajoute.
